# 초보개발자를 위한 한 장짜리 개발 문서 작성법

Tip! 원페이지 작성원칙
1. 쉽고 간결하게 (누가 봐도 이해하기 쉽도록)
2. 사실과 의견 구분 (정확하게 확인된 부분만 사실로)
3. 일기 쓰지 말 것 (자신의 느낌은 쓰지 말것)
4. 디자인에 시간 낭비 하지 마라 (예쁘게 꾸밀 필요 없음)
5. 변경사항 관리 (미루지 말고 바로 바로 관리)

템플릿 소개
- 타이틀 = 프로젝트 이름, 문서 작성자, 작서 날짜, 목차 등
- 프로젝트 설명 = 다른 사라미 이해할 수 있도록 프로젝트를 요약한 설명
- 산출물 = 프로젝트에서 어떤 결과물이 나오는지 정확하게 기재
- 비즈니스, 마케팅 타당성 = 프로젝트를 해야하는 근거, 프로젝트의 정체성
- 리스크 평가 = 프로젝트 중 예상되는 문제와 대응방안 (굉장히 중요!!!)
- 일정/참여 인력 = 몇 명이 언제까지 개발할 것인지, 시간과 인력을 얼만큼 활용할 수 있는지에 대한 계획 
  (Lap Time = 개발장에게 일정, 시간 관리 중요!, 일정에 대한 기준!)
- 기술 설명 = 개념, 동작원리 등에 대한 설명, 글보다 그림을 활용하는 것을 추천 (형식에 구애 x)

정리
- 형식보다 내용이 중요
- 스스로 필요성을 느껴야 한다!
- 당연하게 작성해야 한다는 것을 느낌
- 템플릿 보다는 각 항목의 의미를 중시!
- 왜 항목을 적으라고 했는지에 대해서도 고민!

마무리 네 줄 요약
- 문서작성의 본질은 공유
- 문서작성이 개발의 시작
- 변경사항이 있으면 즉시 업데이트
- 기억보다는 기록
